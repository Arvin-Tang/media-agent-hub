# 《别再只会写 Prompt：AI 工作流的四级进化》开发 Outline

## 基本信息

- 形式：16:9 横屏网页演示 / 可录屏长视频
- 目标时长：3-5 分钟
- 主题：从 Prompt 到 Skill、Hook、Loop 的 AI 工作流进化
- 核心主张：真正会用 AI 的人，不只是会提问，而是会设计 AI 循环

## 章节结构

### 1. Cold Open：聊天框为什么越用越乱

- 步数：3
- 作用：建立问题意识
- 屏幕内容：AI 工具变强 vs 工作流变乱；重复说明格式、流程、避坑；没有结构的聊天框
- 信息池：AI 能力提升、重复提示、状态断裂、临时工隐喻

### 2. Four Layers：四层结构总览

- 步数：3
- 作用：提出全片框架
- 屏幕内容：Prompt / Skill / Hook / Loop 四层结构；四层分别解决什么；临时使用 vs 工作流设计
- 信息池：一次表达、能力沉淀、事件触发、系统循环

### 3. Prompt To Skill：从一句话到一种能力

- 步数：4
- 作用：解释 Prompt 的边界和 Skill 的必要性
- 屏幕内容：一次性任务、高频 Prompt、Skill 说明书、代码审查例子
- 信息池：总结群聊、修 Bug、改文案、每周三次规则、验收标准、禁区

### 4. Review Example：重复 Prompt 的真实成本

- 步数：3
- 作用：把抽象概念落到具体场景
- 屏幕内容：代码审查反复提醒事项；重复输入成本；Prompt 与 Skill 的差异
- 信息池：安全漏洞、边界条件、性能、异常处理、测试覆盖、架构一致性

### 5. Skill Manual：Skill 是一份能力说明书

- 步数：3
- 作用：展开 Skill 的结构与案例
- 屏幕内容：方法说明书；输入/输出/合格标准/必须避免；Superpowers 案例
- 信息池：Markdown Skill、TDD、系统化调试、子代理协作、代码审查、验证后完成

### 6. Discipline：把方法从聊天记录里拿出来

- 步数：3
- 作用：强调 Skill 背后的工程纪律
- 屏幕内容：AI 编程常见失控；可复用流程；提醒 vs 训练
- 信息池：需求未清楚就开工、测试未跑就宣布完成、流程外化

### 7. Manual Call：AI 不该一直等你开口

- 步数：3
- 作用：从 Skill 过渡到 Hook
- 屏幕内容：主动调用的局限；人推任务给 AI；Hook 让 AI 自动进入流程
- 信息池：打开聊天框、复制内容、提醒 AI、事件发生

### 8. Event Entry：事件才是真实工作流入口

- 步数：3
- 作用：解释 Hook 的本质
- 屏幕内容：任务完成通知、新文件归档、上下文摘要；事件入口；自动触发 Skill
- 信息池：文件出现、会议结束、代码提交、用户反馈、系统报错、权限卡住

### 9. Content Loop：把动作串成系统

- 步数：3
- 作用：从 Hook 升级到 Loop
- 屏幕内容：自动动作 vs 系统；内容创作流水线；AI 内容系统
- 信息池：拉群消息、筛 AI 话题、生成洞察、写文章、做漫画、写动画脚本、归档素材

### 10. File System：文件系统接管聊天记录

- 步数：3
- 作用：引入工程化案例并总结变化
- 屏幕内容：OpenSpec / Superpowers / Comet 三层；聊天记录不是工程系统；工作流重设计
- 信息池：WHAT / HOW / WHEN、需求、状态、验证证据、下一步动作

### 11. Action：今天就能开始的小 Loop

- 步数：2
- 作用：收束行动建议
- 屏幕内容：三件小事；最终金句
- 信息池：高频 Prompt、烦人时刻、重复流程、设计循环
