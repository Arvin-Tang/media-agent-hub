# AI 技术名词一本通：Agent、Skill、Harness 到底啥意思？

> 从 LLM 到 Agent，AI 圈的名词轰炸终于有人讲清楚了。

---

## 一、先搞明白：AI 在进化什么？

如果 AI 是一个人的话，它的进化路径大概是这样的：

**阶段一：纯聊天（LLM）** → 你问一句它答一句，像个超级搜索引擎+作文生成器

**阶段二：会查资料（Prompt + RAG）** → 开始会"翻书"了，能基于外部知识回答你

**阶段三：会干活（Agent）** → 不光会回答问题，还会主动帮你完成任务

**阶段四：会组队（Agent + Skill + Harness）** → 多个 Agent 互相配合，各司其职

看不懂？没关系，我们一个一个拆。

---

## 二、LLM：AI 的"大脑"

**全称**：Large Language Model（大语言模型）
**代表产品**：ChatGPT、DeepSeek、Claude

LLM 就是一个训练了大量语料（整个互联网的文本）的超级"大脑"。你给它一段话，它根据训练学到的模式，猜出最合理的下一段话。

**打工人类比**：像一个刚毕业的实习生——知识面广，啥都能聊两句，但没实际工作经验，也没工具。

---

## 三、Prompt：你怎么跟 AI 说话

**中文俗称**：提示词

Prompt 就是你给 LLM 的那段输入。别小看这件事——同一个人问同一个 LLM，换一句 Prompt，答案质量能差十倍。

**比如：**
- ❌ "写个文案" → 随缘产出
- ✅ "你是一个10年经验的文案总监，擅长小红书爆款，请为一个面向25岁女性的轻奢香水品牌写3条小红书标题，每条不超过20字，要求有情绪钩子" → 高质量产出

**趋势**：现在已经进化到 Prompt Chain（提示词链）了——多个 Prompt 串联，让 AI 一步一步思考。

---

## 四、RAG：让 AI 学会"查资料"

**全称**：Retrieval-Augmented Generation（检索增强生成）

LLM 有个致命问题：**它的知识截止于训练那天**。你问它本月的新闻，它一脸懵。

RAG 的解决方案很简单粗暴：
1. 你先去数据库/搜索引擎里找到相关资料
2. 把资料+"用户问题"一起扔给 LLM
3. LLM 基于这些资料回答问题

**打工人类比**：开会时偷偷百度搜索——表面上是自己回答，实际上查了资料。

**你每天都在用**：大多数 AI 搜索产品（Perplexity、秘塔、元宝）用的都是 RAG 架构。

---

## 五、Agent：AI 从"嘴炮"到"实干"

**中文翻译**：智能体 / 代理

Agent 是 LLM 的升级版——**不只回答问题，还能执行任务**。

一个 Agent 的核心要素：
1. **大脑**（LLM）—— 理解任务、做决策
2. **眼睛和手**（Tool/Skill）—— 能看网页、写文件、发邮件、调API
3. **记忆**（Memory）—— 记得上下文，知道你刚才说了啥
4. **规划能力**（Planning）—— 复杂任务分成步骤执行

**打工人类比**：实习生转正了——不光能回答"这个怎么做"，还能直接帮你做完。

**一个 Agent 干活的流程：**
> 你说："帮我订下周二下午2点去上海的机票，预算800以内"
>
> Agent 1️⃣ 调日历查时间 2️⃣ 搜航班比价 3️⃣ 填表单下单 4️⃣ 把行程写到你日历上

---

## 六、Skill：Agent 的"技能包"

**中文俗称**：技能 / 工具

Agent 需要 Skills 才能干活。一个 Skill 就是一段可复用的能力：

- **搜索引擎 Skill** → Agent 能搜网页
- **计算器 Skill** → Agent 能做计算
- **画图 Skill** → Agent 能生成图片
- **发送邮件 Skill** → Agent 能替你发邮件

**打工人类比**：Excel 宏/快捷键——不是人人都会，但会的人效率翻倍。

**关键概念**：Skills 可以排列组合。一个 Agent 装上「搜索 Skill + 写作 Skill + 排版 Skill」，就能自己去公众号写文章。

---

## 七、Harness：Agent 的"调度中心"

**中文俗称**：容器 / 框架

当你拥有了**多个 Agent**，每个 Agent 装上了**不同 Skills**，谁来管它们？

Harness 就是那个"发号施令"的角色：
- 哪个 Agent 该干活了？
- 任务拆成几步？
- Agent 之间怎么交接？

**打工人类比**：项目经理——他不用干活，但他知道谁该干什么、什么时候干完。

**比如 QClaw / OpenClaw：**
> Harness 是这个系统本身，Skills 是各种能力（飞书、浏览器、搜索），不同的 Agent（writer/video-creator/comic）各司其职，Harness 负责调度它们完成一个完整的创作流水线。

---

## 八、一张图总结

```
用户输入 "帮我做个关于AI名词的视频"
│
▼
Harness（调度中心）
│  拆解任务 → 先做脚本 → 再做配图 → 再合成视频
│
├── Agent A（写作手）→ Skill: 搜索 + 文档
│   输出：视频脚本
│
├── Agent B（插画手）→ Skill: 文生图
│   输出：配图素材
│
├── Agent C（剪辑手）→ Skill: 文生视频
│   输出：视频成品
│
▼
交付：一条完整的视频
```

这个链条，就是 2025-2026 年 AI Agent 行业最前沿的工作模式。

---

## 写在最后

AI 技术名词之所以让人头大，不是因为它们本身复杂，而是**太多人把它们包装得高大上**。

记住一个公式：**LLM + Skills + Memory + Planning = Agent，多个 Agent + Harness = 真正的 AI 生产力**。

下次再听到有人说"我们搞了个 Agent 产品"，你就可以问：能拆成几步？谁在调度？装了什么 Skill？

大概率对面会沉默三秒钟 🤫
