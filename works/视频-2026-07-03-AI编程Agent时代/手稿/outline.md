# 《AI 编程进入 Agent 时代》开发 Outline

## 基本信息

- 形式：16:9 横屏解释视频 / 开发者传播向
- 目标时长：3 分钟左右
- 主题：AI 编程从补全代码进入 Agent Mode，开发者能力从写代码迁移到定义任务、审查结果、控制风险
- 核心主张：真正的变化不是 AI 会写更多代码，而是 AI 开始执行研发流程

## 章节结构

### 1. Cold Open：别再只把 AI 当 Tab 键

- 时长：0-15s
- 作用：建立代际差异和危机感
- 内容节奏：从“补全代码”切入，迅速提出“AI 正在执行任务”
- 信息池：高级 Tab 键、Agent Mode、读项目、改代码、跑测试、修报错

### 2. 三阶段演进：补全、对话、Agent

- 时长：15-45s
- 作用：给观众一个清晰框架
- 内容节奏：每阶段一句核心定义，强调从光标后补全到目标驱动执行
- 信息池：代码补全、对话式编程、Agent Mode、问答式工具、任务式代理

### 3. Agent Mode 改变了什么

- 时长：45-90s
- 作用：把概念落到研发流程
- 内容节奏：描述一个跨文件任务从定位到验证的闭环
- 信息池：读仓库、找相关文件、多文件修改、运行测试、根据报错修复、变更说明

### 4. 开发者能力迁移

- 时长：90-135s
- 作用：说明开发者不是消失，而是工作重心变化
- 内容节奏：拆任务、审结果、要证据三连击
- 信息池：目标清晰、边界明确、代码审查、接口约定、安全问题、验证证据

### 5. 小团队红利

- 时长：135-160s
- 作用：说明为什么独立开发者和小团队尤其该关注
- 内容节奏：一个人覆盖更多研发环节，但避免夸张承诺
- 信息池：需求、实现、测试、文档、发布、重复性工作、跨文件工作、研发杠杆

### 6. 风险提醒与结论

- 时长：160-180s
- 作用：建立可信度，避免把 Agent 神化
- 内容节奏：先说风险，再给正确姿势，最后落金句
- 信息池：幻觉、误解项目结构、不存在 API、技术债、权限、安全、回滚、人工审查

## 内容边界

- 不把某个具体产品演示写成确定事实，除非发布前完成来源核验
- 不宣称程序员会被整体替代
- 不编造具体案例、数字或厂商政策
- 对工具能力使用“尝试”“越来越多工具”“方向上”等谨慎表达

## 素材清单

- 无需外部素材
- 可使用代码编辑器、文件树、测试结果、任务清单等抽象界面元素
- 可选补充：已核验的工具官方页面截图、脱敏项目结构示例
