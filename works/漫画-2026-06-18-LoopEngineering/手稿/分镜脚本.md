# 漫画分镜脚本：什么是 Loop Engineering？

**创作日期：** 2026年6月18日
**形式：** 长漫画（6格，三段式叙事）
**主题：** 用漫画讲清楚 Loop Engineering 是什么
**风格：** 轻松教学漫风，有"对比感"

---

## 角色设计

| 角色 | 描述 | 特点 |
|------|------|------|
| **小智** | 爱偷懒但聪明的程序员（代表老方法） | 顶着乱糟糟的头发，戴着黑框眼镜 |
| **老A** | 成熟冷静的架构师（代表新方法） | 头发整齐，永远穿着格子衬衫，旁边有台笔记本电脑 |
| **Agent** | 一个可爱的半透明小精灵 | 圆圆的，会发光，脚底有旋风 |

---

## 分镜详情

### 格 1 — 小智的日常：狂调 prompt

**画面描述：**

一张办公桌上的特写画面。小智的工位仿佛被风暴袭击过——显示器上开了十几个标签页，键盘上散落着零食碎屑，可乐罐子东倒西歪。

小智本人趴在桌前，眼睛布满血丝，一只手攥着鼠标，另一只手揪着自己的头发。他的对话气泡里是一长串：

**"帮我写个 Python 函数……不对，用 Go……等等还是 Python……参数要这样……不对那样……再加个错误处理……"**

显示器上，AI 回复了一行接一行的代码，旁边贴着各种颜色的便签纸：

**"已试 prompt v23" / "v24 也不行" / "v25 接近了" / "v26 换模型试试"**

画面角落里，**老A**端着一杯咖啡路过，停下脚步，看着这一幕露出"我懂的"表情，气泡里一个字：

**"唉。"**

**画面注释 / 小字：**
> 这就是 Prompt Engineering 的日常——你比 AI 还累。

**创作说明：**
- 全画幅特写，视觉冲击力强
- 小智的崩溃状态要夸张
- "v23/v24/v25" 的便签形成视觉节奏，暗示"反复调了二十多次"
- 老A只用出现半个身子+杯子，视线引导到下一格

---

### 格 2 — 老A 的魔法：Agent 自己跑

**画面描述：**

切换到老A的工位——整洁得仿佛展示间。桌上只有一台笔记本，屏幕亮着，旁边放着一杯热茶。

老A靠在椅背上，双手枕在脑后，悠闲地闭着眼微笑。他的工位上有一个大大的**透明气泡框**，框里是一个正在运行的"循环"示意图：

```
          ┌──────────────────┐
          │  🎯 目标：      │
          │  "清洗销售数据"  │
          └────────┬─────────┘
                   │
          ┌────────▼─────────┐
          │  Agent 开始干活   │
          │  → 读取文件       │
          │  → 调用清洗函数   │
          │  → 检查格式       │
          └────────┬─────────┘
                   │ ❌ 格式不对
          ┌────────▼─────────┐
          │  Agent 自动修正   │
          │  → 重新清洗       │
          │  → 再次校验       │
          └────────┬─────────┘
                   │ ✅ 通过
          ┌────────▼─────────┐
          │  ✅ 任务完成      │
          │  输出到成品目录    │
          └──────────────────┘
```

画面右下角，小智探出半个脑袋，目瞪口呆：

**"等等……你的 AI 在自动干活？你都不用动一下？"**

**老A**睁开一只眼，咧嘴一笑：

**"我在设计循环，不是在写 prompt。"**

**创作说明：**
- 两个工位的视觉反差——整洁 vs 混乱
- 循环图要画得清晰易懂，箭头流转方向明确
- 小智的惊吓表情从右下角探出来，有"打破第四面墙"的感觉

---

### 格 3 — 从 prompt 到循环：四代演进

**画面描述：**

画面分成四块，从上到下排列，像进化论那样展示：

**第一块（最旧，暗色调）：**
**"Prompt Engineering — 我教你说话"**
一个放大镜对准对话框，旁边有个人在苦苦打字。小字：*2023-2024*

**第二块（稍微亮一点）：**
**"Context Engineering — 我给你背景"**
一堆文档、PDF、代码库堆在 AI 面前。小字：*2024-2025*

**第三块（更亮，有工具图标）：**
**"Harness Engineering — 我给你工具链"**
AI 手里拿着扳手🔧、锤子🔨等工具，旁边有行代码框。小字：*2025-2026*

**第四块（最新，最亮，暖色发光）：**
**"Loop Engineering — 我给你一个能自己跑的循环"**
一个发光的圆环 ♻️，AI 在里面自动运转，外围标注：目标→行动→观察→修正→完成。小字：*2026.06 — 现在*

四块之间用箭头连接，箭头逐渐变粗变亮。

**小智**站在第四块旁边，指着它，一脸兴奋：

**"2026 年 6 月——Claude Code 之父和 OpenClaw 创始人同时力捧的新范式！"**

**老A**站在第三块和第四块之间，微微一笑：

**"关键不是工具更多了——是 AI 能自己闭环了。"**

**创作说明：**
- 四块区域用渐变色彩区分，从灰暗到明亮
- 每块可以加入"技术标志性事件"的小图标（比如 Harness 区可放 OpenAI Codex Logo）
- 第四块的发光圆环要突出，画大一点
- 小智站在第四块旁边，暗示"这才是现在"
- 整体像"人类进化图"那样的从左到右/从上到下排列

---

### 格 4 — Loop 的五个核心组件

**画面描述：**

画面中央是一个大大的齿轮 ⚙️，由五个叶片组成。每个叶片代表 Loop Engineering 的一个核心组件。齿轮中央有一个发光的 ⚡️ 标志。

五个叶片向外辐射，每个都配有图标和说明：

```
         🎯 目标
      （要达成的结果）
          ⬆️
🔧 工具 ← ⚡️ Loop → 📋 上下文
（能调用的）（齿轮转起来）（背景信息）
          ⬇️
      📊 评估 & 🛑 停止
    （判断好坏 + 何时停）
```

每个叶片旁边有一行大字说明：

| 叶片 | 一句话解释 | 视觉提示 |
|------|-----------|---------|
| 🎯 目标 | 不是"写一个函数"，而是"让销售流程跑通" | 靶心图标 |
| 📋 上下文 | 给 AI 看懂世界的背景信息 | 一本书📖 |
| 🔧 工具 | 文件系统、数据库、API……AI 能用的家伙 | 扳手🔧 |
| 📊 评估 | 自动检查干得好不好 | 对勾/叉号✅❌ |
| 🛑 停止 | 完成了就停，三次不行也停 | 停止标志🛑 |

画面下方，**小智**双手比划，对读者说：

**"你不再写一条条 prompt 了——你把这五个东西搭好，Agent 自己在循环里跑。"**

右上角，**老A**歪着头补充了一句：

**"这五个组件加起来，就是一个『无限迭代的 AI 打工仔』。"**

**创作说明：**
- 齿轮居中的构图，五个叶片对称排列
- 齿轮的齿要做成箭头形状，暗示"转动方向"
- 左侧叶片可用深色，右侧用亮色，视觉上"从左到右越来越亮"
- 小智的手指比划有"搭积木"的感觉

---

### 格 5 — 真实场景：用 Loop 写周报

**画面描述：**

一个真实的应用案例。画面分三块，从左到右水平排列：

**① 输入：小智的原始数据**
一堆截图、聊天记录、会议纪要的碎片飘在空中，小智对着它们一脸无奈。

**气泡：** "我每周都要写周报，素材散得到处都是……"

**② Loop 运转中**
中央是一个发着蓝光的"周报 Agent 循环"。循环内有三个小图标旋转：

```
📥 读取数据 → 📝 提取要点 → ✏️ 生成三个版本
                   ↕️
              小智选择+反馈
```

旁边有 **Agent 小精灵**在循环周围忙碌地飞来飞去。

**③ 输出：成品**
画面右侧飘出三份已写好的周报草稿，标签分别是"详细版"、"精简版"、"数据为主版"。

小智的工位上，他已经靠在椅子上开始喝咖啡了。

**老A** 从旁边路过，点头说了一句：

**"第一次要设定目标、工具、评估标准。之后每周只需要说一句：'帮我跑周报循环。'"**

**创作说明：**
- 三块布局，左→中→右是"输入→处理→输出"的时间线
- 中间的循环图可以做成抽象的半透明齿轮效果
- Agent 小精灵在循环旁飞，增加动态感
- 小智从无奈→震惊→悠闲的表情变化

---

### 格 6 — 总结：从"按回车的人"变成"设计循环的人"

**画面描述：**

画面是上中下三栏布局，像一张信息卡片。

**上半部分：** 一个醒目的对比框

```
左边（暗，灰色调）               右边（亮，暖色调）
🔴 写提示词                     🟢 设计循环
  小智："再改一下……再试一次……"    老A："搭好组件让它自己跑。"
  一次一问                         一次一系统
  人在循环里                      人在循环外
  你比 AI 累                      AI 比你累
```

**中间部分：** 老A和小智并肩站在画面中央，面向读者。小智的表情已经从懵逼变成了"顿悟"。

**小智：** "所以我的工作不再是帮 AI 写 prompt——"

**老A：** "——而是设计一个能自己迭代的系统。"

两人同时看向镜头：

**合：** "这就是 Loop Engineering。"

**下半部分 — 金句落地框（暖橙色背景，白色大字）：**

> **写 prompt 是问问题，设计循环是建系统。**
>
> 2026 年最值钱的能力，是让 AI 自己跑的"闭环思维"。

**创作说明：**
- 上半部分的红蓝对比要像"游戏角色选择界面"那样格调统一
- 中间部分小智和老A的站位要像"英雄集结"的最后定格
- 金句用粗体大字居中，暖橙背景有"点燃"的感觉
- 整体收获感强，像一篇好文章的结尾

## 创作备注

- **生成方式**：happyhorse-t2v (Agnes) 文生图，按格逐格生成 PNG
- **每格尺寸**：1024×768px 横版，后续可拼成长图
- **画风**：扁平卡通风，蓝橙互补色为主色调
- **色彩过渡**：前两格（橙黄→蓝紫）+ 中两格（灰蓝→亮紫）+ 后两格（暖橙收束）
- **文字处理**：出图后后期叠加气泡文字
- **关键对比线索**：小智的"乱" vs 老A的"简"贯穿全篇
