# AI 技术名词一本通——从 Prompt 到 Agent，一文看懂 2026 年最火的 10 个概念

## 引言

2026 年，AI 已经渗透到生活的方方面面。但当你打开一篇技术文章，满眼的 "RAG"、"Agent"、"MCP"、"LoRA"、"Function Calling"……是不是瞬间头大？

别慌。这篇文章把 2026 年最常遇到的 **10 个 AI 技术名词**，用最简单的大白话讲清楚。不求深，只求你下次听到这些词，心里有数。

---

## 1. 大模型（LLM）—— 聪明但"懵懂"的全能助手

GPT、Claude、DeepSeek、Gemini、Qwen……这些名字你肯定听过。它们都叫 **大语言模型（Large Language Model，LLM）**。

**一句话理解**：LLM 就像一个读了全人类知识的天才实习生——知识面极广，但刚入职，不懂你的公司文化和具体流程。

LLM 的核心能力：
- **文本生成**：写文章、写代码、写邮件
- **理解与推理**：读懂上下文，分析逻辑
- **知识问答**：回答问题、解释概念

但 LLM 也有三个致命弱点：
1. **知识截止日期**：训练完就停在这个时间点了
2. **幻觉问题**：不懂也会硬编答案
3. **无法操作外部系统**：只能聊天，不能执行

后面所有技术，本质上都是在**弥补这三个缺陷**。

---

## 2. Prompt（提示词）—— 你跟 AI 说的每一句话

**Prompt** 就是你输入给 AI 的指令。中文叫"提示词"或"指令"。

**核心特点**：临时性、一次性、用完即丢。

同样一个问题，不同的 Prompt 天差地别：

❌ 差 Prompt："写一篇文章"
✅ 好 Prompt："你是一名科技博主，请写一篇 800 字左右的 AI 科普文章，面向完全不懂技术的普通读者，语气轻松幽默，用比喻解释概念"

**Prompt Engineering（提示工程）** 就是研究怎么写好 Prompt 的技术。在 2023-2024 年，这是最火的 AI 岗位之一。但随着模型越来越聪明，对 Prompt 技巧的要求反而在降低——2025 年 DeepSeek 等推理模型的兴起，让"自然说话"反而比"套模板"更有效。

---

## 3. RAG（检索增强生成）—— 给 AI 配一本"参考书"

**RAG = Retrieval-Augmented Generation**（检索增强生成）。

**为什么要 RAG？** 大模型的知识是"背"出来的，考完试就忘了新知识。你问它公司内部的政策、昨天的新闻，它答不出来。

**RAG 的解决思路**：不重新训练模型，而是在你提问时，先去知识库里搜相关的内容，然后把搜到的内容和你的问题一起交给 AI，让它"看资料回答问题"。

**工作流程**：
```
你提问 → 系统去向量数据库搜最相关的内容 → 把内容和问题打包给 LLM → LLM 基于资料回答
```

**典型应用**：智能客服（实时查产品手册）、企业内部知识库、法律条文查询。

**关键组件**：
- **向量数据库**（Vector Database）：把文字变成"向量"，按相似度检索
- **Embedding**：把人类语言转成机器能理解的数学向量
- **Chunk（分块）**：把长文档切成小块再存，方便精准检索

---

## 4. Agent（智能体）—— 能自己干活的 AI

**Agent** 是 2025-2026 年最火的概念。如果说 LLM 是"你说一步，它做一步"，那 Agent 就是"你给一个目标，它自己从头到尾干完"。

**Agent 的四大能力**：
1. **规划（Planning）**：把大目标拆成小步骤
2. **记忆（Memory）**：记住上下文和中间结果
3. **工具使用（Tool Use）**：调用 API、查询数据库、操作软件
4. **行动与观察（Action & Observation）**：执行操作，观察结果，调整策略

**举个栗子**🌰：
- 普通 AI："帮我订一张下周去上海的机票" → 它只会说"好的，请去携程"
- Agent AI："帮我订一张下周去上海的机票" → 它自己打开浏览器，查航班，比价格，下单，然后把确认信息发给你

**Agent 的关键技术**：
- **ReAct 框架**：Reasoning + Acting，思考→行动→观察→再思考的循环
- **Tool Calling / Function Calling**：让 AI 调用外部工具的能力
- **Multi-Agent（多智能体）**：多个 Agent 协作完成复杂任务

---

## 5. MCP（模型上下文协议）—— AI 的"USB-C 接口"

**MCP = Model Context Protocol**（模型上下文协议），由 Anthropic 在 2024 年底提出。

**为什么需要 MCP？** 以前每个 AI 应用连接外部工具都要"私有定制"，就像每个设备都有自己的充电线。MCP 就是给 AI 行业制定了一个**统一标准**——所有的 AI 模型和外部工具都按这个协议对接。

**MCP 的工作原理**：
- **MCP Server**：把工具（文件系统、数据库、API）包装成标准接口
- **MCP Client**（AI 模型端）：按标准协议调用这些接口

**类比**：USB-C 出现之前，你有十几个充电线。USB-C 出现后，一根线充所有设备。MCP 就是 AI 世界的 USB-C。

**截至 2026 年的影响**：几乎所有主流 AI 框架（Claude、LangChain、Vercel AI SDK 等）都已支持 MCP，大大降低了 AI 应用开发的门槛。

---

## 6. LoRA（低秩适配）—— 给大模型"微整形"

**LoRA = Low-Rank Adaptation**（低秩适配）。

**问题**：训练一个大模型成本极高（GPT-4 训练成本上亿美元）。每次为了特定任务重新训练不现实。

**LoRA 的思路**：不动原模型，只给模型加一层"外挂适配器"。

**类比**：大模型是"底妆"（粉底），LoRA 是"局部修饰"（遮瑕笔）。不改底妆本身，只做局部调整。

**应用场景**：
- 把通用模型变成某个领域的专家（医疗、法律、代码）
- 个性化模型风格（写作风格、绘画风格）
- 在消费级显卡上也能微调大模型

---

## 7. Function Calling（函数调用）—— AI 的"动手能力"

**Function Calling** 也叫 **Tool Calling**，是让 LLM 能够调用外部函数/API 的能力。

**为什么重要**：没有 Function Calling 的 AI 只能"动嘴"，有了它 AI 就能"动手"——查天气、订票、发邮件、执行代码。

**工作流程**：
```
用户："帮我查一下北京的天气"
→ LLM 识别意图，发现需要调用 get_weather(city="北京") 函数
→ 系统执行函数，拿到结果
→ LLM 把结果转成自然语言："北京今天晴，25-32°C"
```

**Function Calling 是 Agent 的"手"**——它是 Agent 能调用工具的技术基础。

---

## 8. Fine-tuning（微调）—— 给 AI"补课"

**Fine-tuning** 就是用特定领域的数据，对预训练好的模型做**二次训练**。

**和 LoRA 的区别**：
- **Fine-tuning**：给整个模型补课（改动大、成本高、效果好）
- **LoRA**：给模型加外挂适配器（改动小、成本低、效果尚可）

**什么时候需要 Fine-tuning**：
- 模型需要掌握大量领域知识（医疗术语、法律条文）
- 模型的输出风格需要统一（企业客服语调）
- 模型的推理路径需要符合行业规范

**什么时候不需要**：
- 只是需要查最新信息 → 用 RAG
- 只是临时改个风格 → 调 Prompt 即可

---

## 9. RLTF / RLHF（强化学习）—— AI 的"三观教育"

**RLHF = Reinforcement Learning from Human Feedback**（基于人类反馈的强化学习）。

**核心思想**：AI 学了很多知识，但它不知道什么该说、什么不该说。RLHF 就是**用人来教 AI 价值观**。

**流程**：
1. 让 AI 生成多个答案
2. 人类排序："这个回答好，那个回答不好"
3. AI 从排序中学习，调整自己的行为

**RLTF (Reinforcement Learning from Tool Feedback)**：2025 年兴起的新技术，用工具执行结果（而不是人类）来反馈——AI 写了一段代码，编译通过就是"好"，报错就是"不好"。

---

## 10. 多模态（Multimodal）—— AI 的"五感"

**多模态**指 AI 不仅能处理文字，还能理解**图片、音频、视频、代码**等多种信息。

**经典能力**：
- **文生图**：DALL·E、Midjourney → 输入文字，输出图片
- **图生文**：识别图片内容，生成描述
- **语音交互**：听懂你说的话，用语音回答
- **视频理解**：看懂视频中的场景和动作

**2026 年的趋势**：多模态正在从"能看懂"进化到"能理解"——AI 看一张照片，不仅知道"这是一只猫"，还能理解"猫很放松，阳光很好"这样的情绪和氛围。

---

## 总结：这些名词之间的关系

拿"做饭"来打个比喻：

| 概念 | 厨房里的角色 |
|---|---|
| **LLM（大模型）** | 一把万能菜刀（基础工具） |
| **Prompt** | 菜谱上的临时备注 |
| **RAG** | 做菜时随时翻的食谱书 |
| **Agent** | 自动炒菜机器人 |
| **MCP** | 统一的厨具接口标准 |
| **LoRA** | 给菜刀磨了个专用刀片 |
| **Function Calling** | 机器人"伸手"拿佐料 |
| **Fine-tuning** | 送厨师去进修 |
| **RLHF/RLTF** | 顾客反馈，改进菜品 |
| **多模态** | 厨师会看、会听、会尝 |

每个概念解决一个具体问题，组合起来，就是 2026 年 AI 的全貌。
