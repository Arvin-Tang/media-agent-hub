# 《HermesAgent：会成长的个人 Agent》开发 Outline

## 基本信息

- 形式：16:9 横屏解释视频
- 目标时长：3 分钟左右
- 主题：HermesAgent 与“会成长的个人 Agent / 数字同事”
- 核心主张：下一代 AI 的关键，不只是回答更聪明，而是能长期和你一起工作
- 事实边界：不硬写未经核验的 Star 数、发布时间、团队归属；涉及项目能力时使用谨慎表述

## 内容节奏

### 1. Cold Open：AI 很聪明，但总像第一天上班

- 时长：0-20s
- 作用：建立痛点
- 内容重点：每次都要重新说明身份、项目、格式、禁区；普通 Chatbot 用完就忘
- 观众感受：这就是我每天重复培训 AI 的感觉

### 2. 提出主题：HermesAgent 代表什么方向

- 时长：20-40s
- 作用：从单个项目过渡到趋势判断
- 内容重点：先不强调未核验的热度数字；重点讲它代表的方向：会成长的个人 Agent
- 核心表达：AI 从一次性聊天助手，变成长期协作的数字同事

### 3. 一句话定义：什么叫会成长

- 时长：40-55s
- 作用：给全片框架
- 内容重点：会成长不是魔法，而是记忆、Skill、Loop 三层能力
- 关键词：长期上下文、能力沉淀、工作流触发

### 4. 第一层：记忆

- 时长：55-95s
- 作用：解释为什么 Agent 需要长期上下文
- 内容重点：记住项目、偏好、历史决策、常用格式、禁区；“按上次那个风格”不应每次重新提供材料
- 关键判断：没有记忆，Agent 只是复杂聊天框；有了记忆，才有资格成为个人工作系统

### 5. 第二层：Skill

- 时长：95-140s
- 作用：解释从 Prompt 到可复用能力
- 内容重点：高频 Prompt 应该沉淀成 Skill；Skill 包含输入、输出、步骤、合格标准、禁区、验证方式
- 例子：代码审查、日报、内容创作、文件归档
- 关键判断：会成长的 Agent 不只是记住过去，还会把成功经验变成下次可调用的方法

### 6. 第三层：Loop

- 时长：140-180s
- 作用：解释 Agent 如何进入真实工作流
- 内容重点：真实工作是一连串事件，不是连续聊天；新文件、会议结束、代码提交、定时任务、任务完成都可以成为入口
- 关键判断：Hook 触发动作，多个 Hook 和 Skill 串起来形成 Loop

### 7. 数字同事：为什么不是私人秘书

- 时长：180-215s
- 作用：强化“数字同事”定位
- 内容重点：秘书偏提醒整理；数字同事参与生产，记住反馈，复用经验，把流程越做越顺
- 关键词：长期协作、反馈、训练、复盘

### 8. 现实边界：可控地变聪明

- 时长：215-250s
- 作用：避免过度承诺
- 内容重点：记忆可能会错，Skill 需要验证，自动化越强风险越高，隐私和权限必须可控
- 核心表达：成熟的个人 Agent 不是无限全自动，而是可控地变聪明

### 9. 收束：从使用 AI 到培养 AI

- 时长：250-180s 或结尾 20s 视剪辑压缩
- 作用：总结价值
- 内容重点：过去是提问、等待、复制、修改；未来是训练、反馈、沉淀、复用、触发、复盘
- 结尾金句：普通 Chatbot 帮你完成一次对话，会成长的个人 Agent 帮你养成一套工作系统

## 信息池

- 普通 AI 的痛点：重复交代、上下文断裂、格式反复说明、历史决策丢失
- HermesAgent 方向：长期协作、个人 Agent、数字同事、可培养的工作系统
- 三层机制：记忆、Skill、Loop
- 谨慎表达：可能、倾向于、这类系统通常会、发布前需以官方资料为准
- 风险边界：记忆治理、隐私权限、验证标准、关键动作确认
