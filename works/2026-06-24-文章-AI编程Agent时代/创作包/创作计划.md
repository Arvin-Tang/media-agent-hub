# AI 编程 Agent 时代创作计划

## 定位

- 主题价值：开发者受众最强，适合做系列里的“行业变化”篇
- 目标读者：程序员、技术管理者、独立开发者、AI 编程工具用户
- 核心标题方向：AI 编程不再是补全代码，而是进入 Agent Mode
- 核心钩子：从 Copilot 到 Cursor，AI 编程正在从“帮你写”变成“替你跑完整流程”

## 主表达

AI 编程的真正变化，不是模型会写更多代码，而是工作方式从“对话式生成代码”变成“目标驱动的自主执行”。开发者的核心能力也随之变化：从写每一行代码，转向拆目标、审结果、控风险、设计工作流。

## 推荐产物

### 1. 公众号长文重写

原稿有较强传播感，但需要从“工具新闻合集”升级为“结构化判断”：

- 第一部分：AI 编程三阶段演进
  - 补全代码
  - 对话式编程
  - Agent Mode
- 第二部分：Agent Mode 到底改变了什么
  - 多文件修改
  - 自动测试
  - 自动修复
  - PR / Review 流程
- 第三部分：开发者能力迁移
  - 需求拆解
  - 代码审查
  - 架构判断
  - 测试与验证
  - 安全边界
- 第四部分：小团队机会
  - 一人公司
  - AI 原生研发流程
  - 成本红利
- 第五部分：风险
  - 生成幻觉
  - 技术债
  - 安全问题
  - 过度信任

标题候选：

1. 《AI 编程进入 Agent 时代：程序员的工作真的变了》
2. 《别再把 AI 当高级 Tab 键，真正的变化是 Agent Mode》
3. 《从 Copilot 到 Cursor：AI 编程的分水岭来了》

### 2. 视频脚本

- 形式：3 分钟横屏解释视频
- 推荐结构：
  1. 0-15s：如果你还把 AI 当补全工具，就落后一代了
  2. 15-45s：三阶段演进
  3. 45-90s：Agent Mode 能做什么
  4. 90-135s：开发者能力怎么变
  5. 135-165s：小团队红利
  6. 165-180s：风险提醒与结论

### 3. 漫画/轮播

- 角色设定：
  - 老程序员：还在等 Tab 补全
  - Agent 程序员：把任务拆给 AI 跑
  - 审查官：负责检查 AI 写得对不对
- 分镜方向：
  1. 以前：AI 帮你补全一行代码
  2. 后来：AI 根据需求生成一个函数
  3. 现在：AI 自己改多文件、跑测试、提 PR
  4. 程序员从键盘侠变成任务设计师
  5. 风险：AI 写得快，但错得也快
  6. 金句：未来厉害的程序员，不是写得最快，而是最会让 AI 安全交付

## 发布前事实核验

草稿中这些内容需要核验：

- GitHub Copilot Agent Mode 的具体发布时间与能力边界
