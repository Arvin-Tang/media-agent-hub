# 《别再只会写 Prompt：真正厉害的人已经在设计 Loop》视频 Outline

## 基本信息

- 形式：16:9 横屏解释视频 / 可录屏演示
- 目标时长：3-5 分钟
- 主题：Loop Engineering 方法论
- 核心主张：真正的 AI Native 能力，不只是写好 Prompt，而是把重复任务设计成可运行、可检查、可停止的循环系统
- 注意：本 outline 只规划内容节奏，不写具体动画方案

## 章节结构

### 1. Cold Open：AI 越强，工作流越累

- 建议时长：20-30 秒
- 作用：建立痛点
- 内容节奏：
  - 提出反差：AI 工具越来越强，但很多人用得越来越累
  - 点出重复劳动：背景、格式、规则、避坑、验收标准每次都要重说
  - 转折：问题不一定是 Prompt 不够好，而是工作流没有结构
- 关键信息：重复 Prompt、上下文搬运、人工管理 AI

### 2. Definition：Loop Engineering 一句话定义

- 建议时长：25-35 秒
- 作用：给出核心概念
- 内容节奏：
  - 先否定误解：不是让 AI 多试几次，也不是更长的 Prompt
  - 给出定义：把目标、上下文、工具、评估和停止条件组织成闭环
  - 抛出金句：Prompt 是一次对话，Loop 是一套系统
- 关键信息：目标、上下文、工具、评估、停止条件、闭环

### 3. Old Way vs New Way：人在循环里，还是人在循环外

- 建议时长：25-35 秒
- 作用：建立对比
- 内容节奏：
  - 老方式：人写 Prompt、AI 回答、人判断、人再改 Prompt
  - 新方式：人设计循环，AI 在循环中行动、观察、修正
  - 强调边界：AI 完成任务或在不确定时交还给人
- 关键信息：人不再每一步手推，AI 不再只等指令

### 4. Five Components：Loop 的五个组件

- 建议时长：70-90 秒
- 作用：讲清方法论主体
- 内容节奏：
  - 目标：可执行、可判断，而不是模糊愿望
  - 上下文：背景、资料、历史、偏好、限制条件
  - 工具：文件、搜索、API、测试、格式检查等行动能力
  - 评估：结果是否合格，不能盲目重复
  - 停止条件：完成、预算、失败次数、不确定风险
- 关键信息：好的 Loop 不只是会继续，也要会停止

### 5. Evolution：从 Prompt 到 Context、Harness、Loop

- 建议时长：35-45 秒
- 作用：把 Loop 放进更大的工作流演进
- 内容节奏：
  - Prompt 解决表达
  - Context 解决信息
  - Harness 解决工具调用
  - Loop 解决系统运转
  - 强调它们不是替代关系，而是叠加关系
- 关键信息：Loop 是组织层，不是单一技巧

### 6. Case 1：内容创作 Loop

- 建议时长：35-45 秒
- 作用：让普通创作者理解应用方式
- 内容节奏：
  - 对比临时 Prompt：“帮我写一篇文章”
  - 展开流程：素材输入、筛选选题、选题卡、发布判断、初稿、事实标注、多平台文案、归档
  - 点出人的位置：人负责方向、审核、最终判断
- 关键信息：内容创作从单次生成变成生产流程

### 7. Case 2：AI 编程 Loop

- 建议时长：35-45 秒
- 作用：连接开发者场景
- 内容节奏：
  - 展开流程：读需求、找文件、定计划、改代码、跑测试、根据错误修复、再测试、输出说明
  - 强调验证：测试没跑就说完成，不是好 Loop
  - 强调停止：失败很多次还乱改，也不是好 Loop
- 关键信息：AI 编程的关键是验证循环，而不只是生成代码

### 8. Case 3：个人周报 Loop

- 建议时长：25-35 秒
- 作用：降低门槛，让非开发者也能迁移
- 内容节奏：
  - 读取会议纪要、聊天摘要、任务记录
  - 提取完成事项和阻塞事项
  - 按模板生成多个版本
  - 检查遗漏并等待人选择或补充
- 关键信息：Loop 节省的是认知负担，不只是打字时间

### 9. Decision：什么任务值得做成 Loop

- 建议时长：30-40 秒
- 作用：避免过度工程化
- 内容节奏：
  - 明确：一次性问题用 Prompt 就够
  - 判断条件：高频、重复规则、多步骤、需检查、用文件或工具、出错成本高
  - 总结：Loop 适合高频、重要、可复用的流程
- 关键信息：不要把所有任务都复杂化

### 10. Action：今天设计第一个最小 Loop

- 建议时长：35-45 秒
- 作用：给行动入口
- 内容节奏：
  - 找一个最常重复的 Prompt
  - 写清输入、输出、验收标准、停止条件
  - 给出停止条件例子：三个版本后停、校验通过后停、失败三次交给人、事实不确定就标注
  - 收束金句：写 Prompt 是问问题，设计 Loop 是建系统
- 关键信息：最小 Loop 可以从手动流程开始，不必一步到位自动化

## 节奏备注

- 前 60 秒必须完成“痛点 + 定义 + 对比”，避免概念铺垫过长
- 中段用“五组件”建立方法论，案例只服务理解，不扩散到未经核验的行业事件
- 结尾强调可执行的小 Loop，不渲染不确定的外部来源或未经核验的大会表述
- 全片语气适合公众号/视频号：清楚、克制、有行动感
