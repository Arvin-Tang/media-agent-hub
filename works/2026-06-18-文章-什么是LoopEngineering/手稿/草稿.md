# 别再写提示词了，设计循环吧——2026 年最火的 AI 新范式：Loop Engineering

> 从 Prompt Engineering 到 Loop Engineering，AI 开发逻辑正在被彻底颠覆。

想象一下这个场景：

你是一个程序员。你想让 AI 帮你写一个支付模块。

**以前你会做什么？** 写一长串提示词，反复调试："帮我写一个支付接口，要安全，要高效，支持支付宝和微信……如果不行再加一句'用 Go 语言写'……还是不对，改成 Python……"

你发现自己不是在写代码，而是在当"提示词质检员"。

**现在，Claude Code 之父 Boris Cherny 说了这么一句话：**

> "我不再给 Claude 写提示词了。我写好一个个运行着的循环，让循环去提示 Claude，让它自己搞清楚该做什么。我的工作就是写循环。"

这就是 **Loop Engineering**（循环工程）。

---

## 一、所以 Loop Engineering 到底是什么？

先说结论，再说细节。

**Loop Engineering 的本质是：从"给 AI 写指令"变成"给 AI 设计一套自己跑的系统"。**

传统方式下，你每次都要手动输入提示词 → AI 回答 → 不满意 → 再改提示词 → 再回答……无限循环。

Loop Engineering 的方式是：你设计一个闭环系统，给 AI 一个目标、一些工具、一套评估标准，然后让它自己在循环里迭代，直到任务完成。

**五个核心组件：**

| 组件 | 干什么的 | 举个栗子 |
|------|---------|---------|
| 🎯 明确目标 | 告诉 AI 要达成什么 | "把这个 Excel 的所有数据清洗成标准格式" |
| 📋 上下文管理 | 提供背景信息和历史 | 已有的数据样例、清洗规则文件 |
| 🔧 可调用工具 | AI 能用的工具 | 文件读写、数据库查询、API 调用 |
| 📊 产出评估 | 判断干得好不好 | 自动跑测试、检查格式是否符合规范 |
| 🛑 停止标准 | 什么时候停下来 | 全部完成、或尝试 3 次仍失败 |

五者组合，AI 不再是"问一句答一句"的答题机，而是进入了一个 **"思考 → 行动 → 观察结果 → 再思考"** 的闭环。

---

## 二、从 Prompt → Context → Harness → Loop：Agent 工程的四次范式跃迁

Loop Engineering 不是凭空冒出来的。它是 Agent 工程演进的自然结果：

```
第一代  Prompt Engineering      →  我教你说话
第二代  Context Engineering     →  我给你背景
第三代  Harness Engineering    →  我给你工具链
第四代  Loop Engineering       →  我给你一个能自己跑的循环
```

每次跃迁，都是在重新定义人和 AI 的协作边界。

**第一代：Prompt Engineering**（2023-2024）——你花 80% 的时间想怎么写提示词。核心能力：写 prompt。

**第二代：Context Engineering**（2024-2025）——你发现提示词不管用了，开始精心设计上下文、文档、示例。核心能力：给 AI 塞信息。

**第三代：Harness Engineering**（2025-2026）—OpenAI Codex 团队用 3 个工程师在 5 个月内生成了 100 万行生产级代码。核心不再是"写什么"，而是"怎么把工具链串起来"。核心能力：搭积木。

**第四代：Loop Engineering**（2026 年 6 月—）——OpenClaw 创始人 Peter Steinberger 在 ClawCon 2026 大会上正式提出：**"别再搞 coding agent 了，去设计能提示人的 agent。"** Agent 从被动执行工具变成了主动引导人类的智能循环系统。核心能力：设计循环。

---

## 三、为什么是 2026 年 6 月突然火起来？

两件事在同一个月发生：

**1. Claude Code 之父 Boris Cherny 的演讲**

他在 Acquired Unplugged 开发者大会上说："我的工作就是写循环。" 这句话在开发者社区迅速传开。他说他不再手动调 prompt 了，而是设计一个让 Claude 自己跑的循环系统。这个系统包含明确目标、工具集、评估标准、容错机制——然后他就能去喝咖啡了。

**2. OpenClaw 创始人 Peter Steinberger 在 ClawCon 2026 的发言**

Steinberger 的视角更大胆。他说：**"别再做 coding agent 了——AI 自己就会写代码。你应该设计的是能提示人类的 agent。"**

这意味着什么？AI 不再是你的工具，而是你的"引导者"。你从"按回车的人"变成了"设计循环的人"。

两件事叠加，Loop Engineering 从一个小众概念，一夜之间成为 AI 开发圈的头号热词。

---

## 四、这对你意味着什么？

### 如果你是开发者

你不需要再花时间调 prompt 了。你需要学的，是**设计一套能让 AI 自主迭代的系统**。这包括理解五个组件、一个状态机，以及如何把复杂任务分解成循环能处理的粒度。

2026 年最值钱的能力不再是"会写代码"，也不是"会写提示词"——而是**会设计循环**。

### 如果你是AI使用者

你可能不写代码，但 Loop Engineering 的逻辑同样适用。

你想用 AI 生成每周的行业简报。与其每周手动写一遍 prompt，不如设计一套流程：告诉 AI 你的关注领域 → 它自己搜索 → 自己筛选 → 自己排版 → 自己发给你。你只需要检查结果，不满意就调整规则。

这就是你的"个人 Loop"。

### 行业现状

2026 年 6 月，Loop Engineering 还处于概念爆发期。已经有公司跑了近 3000 个 Agent 在线执行任务。但真正的问题是——**有多少人理解了"设计循环"和"写提示词"不是同一个维度的事？**

答案：还不多。

这意味着，现在入手，你就是第一批。

## 总结

- **Loop Engineering** = 设计让 AI 自主迭代的闭环系统
- **从写提示词 → 设计循环**，是 Agent 工程的第四次范式跃迁
- **两件事引爆热度**：Claude Code 之父 + OpenClaw 创始人同时力捧
- **核心能力**：不再是 prompt，而是用工具、目标、评估、停止条件搭循环

> 写提示词 vs 设计循环
>
> 提示词是**问问题**，循环是**建系统**。
